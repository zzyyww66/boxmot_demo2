## 7. 改进版 ByteTrack 的优势与代价

- `ByteTrack-Tuned` 仍然是当前 SOMPT22 单数据集榜单里的 HOTA / MOTA / DetA 最优配置：HOTA `54.313`、MOTA `66.681`、DetA `55.026`。
- `ByteTrack-DualTuned` 则拿到了当前表中的最高 RHOTA (`57.772`)、最高 IDF1 (`68.150`) 与最高 AssRe (`61.234`)，说明它更像一套兼顾跨数据集泛化的统一配置。
- `ByteTrack-Imp` 仍然是身份稳定性最强的默认版：IDSW `747`、IDs `766` 都是全场最低，适合优先控制切换和碎裂的场景。
- 从 `Imp` 升级到 `DualTuned`，HOTA / MOTA / IDF1 分别提升 `+0.694 / +0.502 / +1.136`；但代价是 IDSW `+65`、Frag `+80`、Association FPS `-9.7`。
- 从 `Imp` 升级到 `Tuned`，HOTA / MOTA 进一步略高于 `DualTuned`；但 `DualTuned` 反过来在 IDF1 / AssRe 上更占优，因此两者的定位已经出现分化。
- 速度方面，`ByteTrack-Imp` / `ByteTrack-Tuned` / `ByteTrack-DualTuned` 分别达到 `201.2 / 196.0 / 191.5 FPS`，依然明显快于 `BotSort`、`BoostTrack`、`DeepOcSort`、`HybridSort` 与 `StrongSort`。
- 绝对速度冠军仍然是 `OcSort` (`252.7 FPS`)，它也保持了最高 `LocA` (`82.707`) 与最高 `MOTP` (`80.266`)；但综合精度仍落后于最优 ByteTrack 配置。
- `BoostTrack` 仍然保持全场最高 AssA (`55.178`)；`HybridSort` 仍然保持最高 `CLR_Re` (`79.486`) 与最低 `CLR_FN` (`109,933`)；`StrongSort` 依旧是最慢的关联器 (`6.4 FPS`)。

## 10. 结论

- 如果目标是“只看 SOMPT22 单数据集的 HOTA / MOTA 最优”，推荐直接使用 `boxmot/configs/trackers/bytetrack_sompt22_tuned.yaml`。
- 如果目标是“希望继续兼顾跨数据集泛化，同时在 SOMPT22 上拿到更高 IDF1”，推荐使用 `boxmot/configs/trackers/bytetrack_dual_tuned.yaml`。
- 如果目标是“尽量稳住 ID、尽量少切换、同时保持高速度”，推荐使用 `boxmot/configs/trackers/bytetrack_improved.yaml`。
- 如果目标是“纯速度优先”，`OcSort` 仍然是最快的关联器；但在当前 SOMPT22 人类场景上，它并不是综合精度最优。
