## 3. 执行方式与缓存复用

本轮评测使用的统一命令模式如下:

```bash
uv run python -m boxmot.engine.cli eval \
  yolov8m_pretrain_crowdhuman \
  osnet_x0_25_msmt17 \
  <tracker> \
  --source MOT17-full \
  --classes 0 \
  --device 0 \
  --project /root/autodl-tmp/boxmot_demo2/runs_mot17_tracker_compare_20260320_035647 \
  --exist-ok \
  --verbose \
  [--tracker-config <yaml>]
```

缓存复用情况:

- 首个运行的 `ByteTrack-Default` 负责生成 MOT17 的共享 dets/embs 缓存，因此 `CacheSkips=0`
- 其余 `10` 组结果全部复用同一份缓存，均表现为:
  - 日志中出现 `21` 次 `Skipping ... (cached complete...)`
  - 随后出现 `No sequences to process (all cached or no images).`
- 因此后续对比只重新执行 tracking + TrackEval，不重复跑 detector / ReID 前处理

### ByteTrack 结果解读

- `Default` 和 `Original` 保留了更高的 `MOTA / DetA / 速度`，但身份相关指标明显偏弱
- `Improved` 把 `IDSW` 从 `1848` 降到 `906`，把 `IDs` 从 `2772` 降到 `1206`，说明改进后的 zombie rescue / ReID 约束对身份稳定性提升非常明显
- `SOMPT22-Tuned` 是 ByteTrack 系列里 `HOTA / IDF1 / AssA` 最好的配置，综合身份质量最强
- `Dual-Tuned` 在 ByteTrack 改进系配置里给出了更均衡的 trade-off:
  - 相比 `Improved`，`HOTA +0.205`、`MOTA +1.375`、`IDF1 +0.947`
  - 同时还略快于 `Improved`，`Assoc FPS 310.1 vs 307.5`
- 如果把 `Default` 当作速度/召回基线，那么:
  - `Improved` 带来 `HOTA +2.476`、`IDF1 +4.386`、`IDSW -942`
  - `SOMPT22-Tuned` 带来 `HOTA +3.084`、`IDF1 +6.298`、`IDSW -912`
  - `Dual-Tuned` 带来 `HOTA +2.681`、`IDF1 +5.333`、`IDSW -843`

## 6. 结论与建议

- 如果目标是本轮 MOT17 上的综合最优精度，`BotSort` 是当前最佳，`HOTA / MOTA / IDF1` 都是全表第一
- 如果目标是更强的关联稳定性和更少的身份切换，`BoostTrack` 的 `IDSW=594` 最低，`AssA=58.070` 也处于全表第一梯队
- 如果目标是纯关联速度，`OcSort` 最快，达到 `368.5 FPS / 2.71 ms`
- 如果只看 ByteTrack 家族:
  - 追求最佳身份质量，优先选 `ByteTrack-Tuned`
  - 追求更均衡的 `精度 / 速度 / MOTA` 折中，优先选 `ByteTrack-DualTuned`
  - 追求最少 `IDSW / IDs`，优先选 `ByteTrack-Imp`
- 从本次 MOT17 结果看，当前 ByteTrack 改进版已经明显优于 `default / original` 的身份表现，但在综合榜首位置上仍略逊于 `BotSort`
